Apolla Capture for Outlook

Capture any open email into your Viktora Apolla corpus with one ribbon-button click. Works on Outlook for Mac and Outlook for Windows (any account type, including Gmail-OAuth-connected personal Outlook).

Install (5 minutes)

Step 1 — copy this manifest URL:

https://rosscantrell.github.io/apolla-outlook-addin/manifest.xml

Step 2 — sideload into Outlook:

  1. Open Outlook. Open any email (the ribbon needs an open message to show the install path).
    If you're on Outlook for Mac with a Gmail-OAuth account, this works the same — verified empirically during WP-OCR-01 Phase E.
  2. In the email's toolbar, click Get Add-ins (or "Add-ins" → "Get Add-ins" depending on Outlook version).
    On Outlook for Mac you may need to click "More" (…) first to surface "Get Add-ins". On Outlook for Windows it's typically in the Home ribbon.
  3. In the dialog: My Add-ins → scroll to Custom Add-ins → Add a custom add-in → Add from URL…
  4. Paste the manifest URL from Step 1. Click OK, then Install, then accept the "untrusted source" prompt (it's not on AppSource yet; that's tracked as FN-ADDIN-05).
  5. The Apolla ribbon group with Capture + Configure buttons appears on any open message.

First-time setup

  1. Click Configure on the ribbon. A task pane opens.
  2. Paste your Apolla bearer token (the same value as INGESTION_API_KEY on your Apolla server).
    If you don't have one yet: on the machine running schema-browser, run openssl rand -hex 32 and use that value for both the server's INGESTION_API_KEY env var and this field.
  3. Set the base URL — default http://localhost:3001 for local-dev pilots.
  4. Click Test connection. You should see "Apolla server reachable…". If you get a CORS error, check that ALLOWED_WEB_ORIGINS=https://rosscantrell.github.io is set on the Apolla server (per D-ADDIN-15).
  5. Click Save. Token is persisted via Office's roamingSettings (encrypted; syncs across your Outlook installations).

Capturing an email

  1. Open any email.
  2. Click the Capture ribbon button.
  3. Watch for the notification banner above the message:
    • Captured: <subject> | terms: N — new ingest succeeded.
    • Already captured: <subject> | terms: N — same email content was captured before; idempotent re-POST.

Troubleshooting

"Apolla not configured" notification

Click Configure and finish the setup. The Add-in won't POST without a token.

"Apolla auth failed — re-run Configure"

Token doesn't match the server's INGESTION_API_KEY. Re-paste the value in Configure and Save.

"Apolla server unreachable at <base-url>"

Server is down, the URL is wrong, or your network blocks it. Verify schema-browser is running and the URL matches.

"Apolla returned HTTP 4xx/5xx"

Open the browser console (Outlook's "Inspect Element" → Console tab) for the full response. Most likely a CORS misconfiguration or a server-side validation error on the payload.

Configure pane won't open

Some Outlook versions cache the manifest. Remove + re-add the Add-in from the same URL. If the pane still won't open, check that configure.html is reachable directly: open in browser.

Privacy posture. The Add-in only POSTs email content to the Apolla server URL you configure. It does not send telemetry, analytics, or any other data anywhere else (D-ADDIN-12). Bcc recipients are deliberately not captured (D-ADDIN-09). Attachment metadata (name/type/size) is captured but attachment file bodies are not downloaded or sent (D-ADDIN-08).
